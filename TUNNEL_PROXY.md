## 🔧 代理配置方式

### 1. 直接代理模式

**配置**
```json
{
  "proxy": {
    "enabled": true,
    "type": "direct",
    "url": "http://127.0.0.1:7890"
  }
}
```

**工作流程**
```
配置加载 
  ├─ 下载代理：http://127.0.0.1:7890 → yt-dlp 
  └─ 隧道代理：http://127.0.0.1:7890 → axios

视频信息获取 → B站API（通过代理）
字幕下载 → 字幕服务器（通过代理）
视频下载 → CDN（通过代理）
```

**适用场景**
- ✅ 本地 Socks5/HTTP 代理服务（如 Clash）
- ✅ 局域网代理
- ✅ 企业代理网关

### 2. 私密代理模式（推荐用于跨境）

**配置**
```json
{
  "proxy": {
    "enabled": true,
    "type": "private",
    "apiUrl": "https://dps.kdlapi.com/api/getdps/?secret_id=xxx&signature=xxx&num=1&sep=1",
    "username": "d2072651992",
    "password": "1ngjc7uy"
  }
}
```

**工作流程**
```
配置加载（私密代理模式）
  │
  ├─ 调用代理API获取动态IP：dps.kdlapi.com
  │  └─ 获取响应：1.2.3.4
  │
  ├─ 构建认证URL
  │  └─ http://d2072651992:1ngjc7uy@1.2.3.4/
  │
  ├─ 下载代理：传递给 yt-dlp --proxy 参数
  └─ 隧道代理：传递给 axios proxy 配置

视频信息获取 → B站API（通过认证代理）
字幕下载 → 字幕服务器（通过认证代理）
视频下载 → CDN（通过认证代理）
```

**适用场景**
- ✅ 付费代理服务（快代理、讯代理等）
- ✅ 动态代理IP池
- ✅ 按流量计费的代理服务
- ✅ 需要自动轮换IP的场景

## 📊 双层代理的工作模式

### 模式1：直接代理
```
所有请求 → 单一代理服务器 → 目标服务器
          (HTTP/SOCKS5)
```

### 模式2：私密代理
```
视频下载请求 ──→ 认证代理IP1 → CDN
API请求     ──→ 认证代理IP2 → B站API  (同时从API池获取多个IP)
```

## 💡 使用建议

### 选择直接代理的情况
- 已有本地代理服务（强烈推荐对国内用户）
- 使用公司/学校的代理网关
- IP池稳定性要求较低

### 选择私密代理的情况
- 需要频繁跨域访问
- IP被标记需要轮换
- 对速度和稳定性要求高
- 需要保护真实IP

## 🔐 安全特性

1. **认证信息隐藏**
   - 日志中显示：`http://***:***@<IP>/`
   - 完整认证URL仅在传递时使用

2. **代理失败降级**
   - 代理获取失败 → 自动尝试无代理模式
   - 不中断整个分析流程

3. **环境变量支持**
   - 支持 `BILI_COOKIE` 环境变量
   - 与代理配置无冲突

## 🚀 性能优化

**下载加速**
- 代理通常位于主流运营商节点
- 直连 CDN 速度往往比代理更快
- 代理最适合用于 API 请求（带宽占用小）

**推荐方案**
```json
{
  "proxy": {
    "enabled": true,
    "type": "private",
    "apiUrl": "https://...",  // 只用于API请求
    "username": "...",
    "password": "..."
    // 下载时系统会自动优化路由
  }
}
```

## 📞 故障排查

### 问题1：代理获取失败
```
[代理] 获取代理IP失败: 请求超时

原因：API服务器不可达
解决：检查 apiUrl 是否正确
```

### 问题2：认证失败
```
[下载] 使用代理: http://***:***@<IP>/
[下载 stderr] 代理认证失败

原因：用户名或密码错误
解决：检查 username 和 password 配置
```

### 问题3：隧道代理不生效
```
[代理] 使用隧道代理获取视频信息
[错误] B站API请求失败

原因1：代理被限制了特定端口
解决1：联系代理服务商开放端口

原因2：B站检测到代理请求
解决2：增加请求头或更换代理IP
```
